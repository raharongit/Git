5. How to reset a file from staging area

git reset filename

To remove a file or files from staging area.

6. How to see the history

git log

It remembers all the changes we've committed so far, in the order we committed them.

7. More information of history

git log --summary

To see more information for each commit. We can see where new files were added for the first time OR where files were deleted.

It's good overview of what's going on the project.

8. Remote Repo

Go to Github website and create a new empty repository.

To push our local repository to the Github server, we'll need to add a remote repository.

git remote add remotename repo_url

Here remotename is a nickname of this project so generally we call it origin OR it's depend on you whatever you want to call it. Git doesn't care what you name your remotes, but it's typical to name your main one origin.

repo_url is the url of your new empty repository on Github.

9. Pushing Remotely

git push -u remotename master

The push command tells git where to put our commits when we're ready.

We are pushing our local changes to our repository on Github.

Default local branch name is master.

-u tells git to remember the parameters, so that next time we can simply run git push and git woll know what to do.
