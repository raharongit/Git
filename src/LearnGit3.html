10. Pulling Remotely

Let's pretend some time has passed. We've invited other people to our Github project who've pulled your changes, made their own commits and pushed them.

We can check for changes on our Github repository & pull down any new changes by running this command:-

git pull remotename master

11. How to see the Differences

git diff

Run this command before add the file into local repository.

It shows unstaged differences.

If we want the diff of our most recent commit, which we can refer to using HEAD pointer.

git diff HEAD

HEAD is pointer that holds your position within all your different commits.

By default HEAD points to your most recent commit, so it can be used as a quick way to reference that commit without having to look up the SHA.

12. Staged Differences

diff is looking at changes within files that have already been staged.

You want to try to keep related changes together in seperate commits.

Using git diff gives you a good overview of changes you have made & less you add files or directories one at a time & commit them separately.

git diff --staged

To see the changes we just staged. (We stages differences)

13. Some Important Things

git reset it does the file unstaged but it will not remove it permanently. (We can add it later)

Files can be changed back to how they were at last commit by using the command:-

git checkout --filename

get rid off all the changes since the last commit for a particular file.

git checkout seems to work fine without it(--).

It's simply promising the command line that there are no more option after that(--).

This way if you happen to have branch name filename, it will still revert the file, instead of swithching to the branch of same name.
