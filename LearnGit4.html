14. Branching Out

When developer are working on a feature or bug they'll often create a copy (aka branch) of their code they can make separate commits to.

Then when they're done they can merge this branch back into their main master branch.

We want to remove some files from main branch or add some files into main branch so let's create a branch called clean_up.

Where we'll do all the work:-

git branch clean_up

15. Switching Branches

Let's type this command git branch

You will find two branches one is main master & other one is newly created clean_up branch.

Now we can switch branches using this command:-

git checkout clean_up

So now you are in clean_up branch.

Some Important Things

git checkout -b branch_name

To checkout & create at the same time.

This is the same as doing:-

git branch branch_name And git checkout new_branch

16. Remove Files

git rm '*.txt'

It will remove all the txt files.

Remove all the things:-

Removing one file is great and all, but what if you want to remove an entire folder? You can use the recursive option on git rm:-

git rm -r folder_name

This will recursively remove all folders and files from the given directory.

If you happen to delete a file without git rm you will find that you still have to git tm the deleted files from the working tree.

You can save this step by step using the -a option on git commit

Which auto removes deleted files with the commit.

git commit -am "message"

17. Preparing to merge

Are you ready to merge?

Okay! First come in the branch(In which do you want to change or merge).

Let's pretend you want to merge all the changes of clean_up branch to master branch.

Then first come on in master branch through this command:-

git checkout master

Now you are in master branch. Right?

Then take the data from other existing branch then do this:-

git merge clean_up
