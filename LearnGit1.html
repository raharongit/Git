1. To initialize a git repository

git init

This command is used to initialize a git repository.

Here .git directory is hidden here. It contains all sort of directories files inside it. We'll rarely ever need to do anything inside here but it's the guts of Git, where all the magic happens.

When we make a repo then it's always inside in .git

2. git status

What's changes happen in repo. We can say what's changes have been made by user in repo.

3. Remember these things

Staged:- Files are rady to be commited.

Unstaged:-Files with changes that have not been prepared to be committed.

Unstaged:- Files aren't tracked by git yet. his usually indicated a newly created file.

Deleted:- File has been deleted and is waiting to be removed from git.

4. How to add Files in repository

git add --all or git add -A or git add -A . Add all files

git add filename Add a particular file

git add *.txt Add all the txt files in current directory

git add docs/*.txt Add all the txt files in docs directory

git add docs/ Add all the files in docs directory

git add '*.txt' orgit add "*.txt" Add all the txt file of whole project

Some important things

git add -A or git add --all Stages All

git add . Stages new & modified, without deleted

git add -u Stages modified & deleted, without new

git commit -a It means git add -u and git commit
